## 3. Biggest synthesis decisions (the 3–4 that mattered)

1. **Scrubbed ALL asserted reference links; rendered Dario/Fable 5/OpenAI as HUMAN-VERIFY look-up anchors.** Codex and Grok both fabricated the Dario title ("The Adolescence of Technology"); Codex additionally minted a Fable 5 URL and asserted the OpenAI URL as a live link. Per the landmine, every reference is a *go-read-it* anchor a human verifies pre-send. The spec names them and flags exact-title/URL verification — it does NOT summarize them from memory.

2. **Adopted Codex's positioning paragraph + Codex's timeline rows, but fenced them with Claude's honesty-firewall caption.** Codex had the best *structural* ideas (yes-vs-later, "the system is the product," the dense timeline) but the worst *compliance*. The merge takes Codex's information architecture and wraps it in Claude's truth-discipline so the "came faster" device stays welded to the build lane only.

3. **Took Claude's per-client blocks wholesale (especially 008/RM).** Only Claude's 008/RM blocks satisfy all three price landmines at once: full Ascend anchor present, cost-of-assembly stack present, no client's own discounted number, proof-first trigger language, zero FOMO. Codex's are close; Grok's leak a rate conversation into ASAP (entry tier) which is a tier-voice violation. Claude's are adopted with Codex's concrete-action verbs stitched in.

4. **Stitched Gemini's "you own the asset" ownership metaphor into §4.** It's the one image none of the others had that makes "building now" tangible for a non-technical owner — a custom system is *land you own*, it compounds, it doesn't take sick days. Trimmed of Gemini's longer abstract run-up so it doesn't bloat FK or word count.
